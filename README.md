# Hi, I'm Zach

I'm a mostly self-taught developer working in field technical services. Writing software started as a way to solve problems at work, and somewhere along the way it became something I do because I genuinely enjoy it.

Most of what I've learned came from building things rather than studying them. That means there are gaps, and I don't always know where they are until I run into them. It also means that everything here exists because it needed to, or because I wanted to find out if I could build it, not because a tutorial walked me through it.

---

## How I Work

The habits that have made the biggest difference aren't the flashy ones. Projects start with a planning phase before a coding phase. Structure is intentional from the beginning rather than retrofitted later. Type annotations, modular organization, structured logging, separation of concerns, these are things I've actively adopted and apply consistently, not things I reach for when I remember.

I also try to be honest about where I am. Not every problem I've solved was solved cleanly the first time, and the codebases here reflect real decisions made under real uncertainty, documented as they happened. The goal isn't to present finished work, it's to show how I think and what I'm working toward.

## Stack

Primarily Python, with Ruby on Rails for web projects. SQLite for personal projects, PostgreSQL where it makes more sense. I maintain my own self-hosted infrastructure, a Raspberry Pi running Void Linux with nginx, Cloudflare, and Tailscale, which has been its own education in how the pieces of a production environment actually fit together.
